Cicero (for Monocle 3)¶

元URL: https://cole-trapnell-lab.github.io/cicero-release/docs_m3/

  • Ciceroは Monocleをベースにした scATAC-seq解析のRパッケージです。特に、ゲノム上のco-accessibilityを調べることによりゲノム中のcis-regulatory interactions（エンハンサープロモーター相互作用など）を予測することを目的としています。
  • Ciceroの主な目的は、scATAC-seqデータを用いて、核内で物理的に近接している可能性が高いゲノム領域を予測することです。これは、エンハンサープロモーターペアを特定したり、ゲノム領域のシスアーキテクチャーの全体的な構造を把握するために使用されます。
  • scATAC-seqデータは疎であるため、データ中の様々な技術的要因をロバストに補正するためには、類似性によって細胞を集約する必要があります（pseudo-buld）。
  • Ciceroは、 "Cicero co-accessibility score"を提供します。これは-1～1の値をとり、ユーザが指定した距離以内に存在するピークペア各々に対して定義され、高い値ほど co-accessibilityが高いことを意味します。
  • 細胞のクラスタリング、軌道解析、differential accessibility analysis も可能です。
  • 現状ではmonocle2対応のものとmonocle3対応のものがあり、互換性がないので注意。ここではMonocle3ベースのCiceroを使います。

load data¶

  • CiceroはCellDataSet (CDS)オブジェクトにデータを保持します。このクラスはBioconductorにある ExpressionSetクラスから派生したもので、マイクロアレイ実験で用いられていた形式です。
  • CDSオブジェクトの生成方法は Monocle を参照してください。
  • CiceroではCDSオブジェクトの fDataに遺伝子の代わりにピークを格納している。具体的には、chr1_10390134_10391134 の形のピーク情報を必要とします。
  • Ciceroの make_atac_cds 関数は入力としてdata.frameまたは sparse matrix形式のファイルへのパスを受け取ります。
  • 具体的には、このファイルは3つの列を持つタブ区切りのテキストファイルでなければなりません。ファイルにはヘッダ行を持たないようにしてください。
    • 最初の列は、"chr10_100013372_100013596 "という形式のピーク座標
    • 2番目の列は細胞名
    • 3番目の列は、そのピークと重なる各細胞のリード数を表す整数です。

Constructing cis-regulatory networks¶

  • scATAC-seqデータは極めて疎なので、co-accessibility scores正確な推定には、類似した細胞を集約してより密度の高いカウントデータを作成する必要があります(pseudo-buld)。
  • CiceroはこれをkNNアプローチを用いて行い、細胞のマージされたセットを作成します。kNNはtSNEやDDRTreeなどを用いて次元削減されたマップ上で行われます。
  • 次元削減方法はどのようなものでも構いませんが。ここではtSNE と DDRTree の 2 つのバージョンを示します。これらの次元削減方法はいずれもMonocleから入手可能です（Ciceroによって読み込まれます）。
  • 次元削減された座標マップができたら、make_cicero_cds関数を使用して集約された CDS オブジェクトを作成することができます。make_cicero_cdsへの入力は、入力 CDS オブジェクトと次元削減された座標マップです。
  • 座標マップ reduced_coordinates は、data.frame または行列形式で、行名が CDS の pData テーブルの細胞 ID と一致している必要があります。reduce_coordinates の列はreduction dimension オブジェクトの座標にする必要があります。

Visualizing Cicero Connections¶

  • Ciceroには plot_connectionsと呼ばれる共同アクセシビリティを可視化するための一般的なプロット関数が含まれています。この関数はGvizフレームワークを使用して、ゲノムブラウザスタイルのプロットを行います。インタラクションの可視化には Sushi R を使います。
  • plot_connectionsには多くのオプションがあり、いくつかはAdvanced Visualizationセクションで詳しく説明されています。
  • 以下では、遺伝子もプロットできるように、オプションの gene_model データを入れています。

Comparing Cicero connections to other datasets¶

  • Ciceroで得られたインタラクションを他のアッセイで得られた類似のデータと比較することは有用である。
  • 例えば、Ciceroの出力とChIA-PET dataを比較したい場合があります。
  • これを行うために、Ciceroには compare_connections という関数があります。この関数は、接続ペアのconns1とconns2の2つのデータフレームを入力として取り、conns1からconns2で見つかった接続の論理ベクトルを返します。
  • この関数での比較はGenomicRangesパッケージを使用して行われ、比較に傾きを許容するためにそのパッケージのmax_gap引数を使用します。

例えば、Ciceroの予測値を(メークアップされた)ChIA-PET接続のセットと比較したい場合は、以下のように実行します。

  • さらに、Ciceroのプロット機能には、データセットを視覚的に比較する方法があります。

  • これを行うには、 comparison_track 引数を使用します。比較データフレームには、"coaccess"と呼ばれる最初の2つのピーク列を超えた3番目の列が含まれている必要があります。

    • これは、プロット関数がプロットされた接続の高さを決定する方法です。
    • これは、ChIA-PETのライゲーション数のような定量的な尺度であってもよいし、単に1の列であってもよい。

  • プロットオプションの詳細は、マニュアルページの ?plot_connections とAdvanced Visualizationセクションを参照してください。
